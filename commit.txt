feat(cli): add install-hooks command and align build/snapshot defaults

- New `metabokg install-hooks` command installs a unified pre-commit hook
  that runs quality checks, rebuilds CodeKG, and captures snapshots for
  CodeKG, MetaboKG, and DocKG atomically before each commit.
- `metabokg build` default flipped: `--wipe` is now opt-in (keep existing
  data by default), aligning with `codekg build` behavior.
- Snapshot version auto-detected via `importlib.metadata` — hooks no
  longer need to grep pyproject.toml for the version string.
- `.pre-commit-config.yaml` made portable (relative poetry paths) with
  broader hidden-dir excludes (`^\.[^/]+/`) for detect-secrets and
  check-added-large-files.
- Add MetaboKG analysis report and pre-commit refactor notes.
